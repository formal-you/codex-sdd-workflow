# Process Summary

本文件用于维护长期流程摘要。它不替代 `progress.md`，而是负责把“当前做什么、最近做完了什么、旧任务去了哪里”说清楚。

在填写时，用 `[ ]` 标记未完成、待推进或待确认事项，用 `[x]` 标记已经完成或已经确认的事项。

- [ ] 当前判断：

## 当前任务板

- [ ] active parent task：
  - [ ] `tasks/active/TASK-XXX-sample.md`
- [ ] active subtasks：
  - [ ] none

## 近期完成

- [ ] `tasks/history/TASK-XXX-sample.md`
  - [ ] 用一到两行总结已完成内容

## 归档规则

- [x] `progress.md` 只记录当前阶段和 handoff，不承担完整历史
- [x] 当 task 或 subtask 完成且不再活跃时，移入 `tasks/history/`
- [x] 归档前，必须在 `progress.md` 留下 next-step entry、创建下一条 active task，或在 full profile 下记录 backlog item
- [x] 归档后，在这里补一条一到两行的摘要
- [x] 新 session 启动时，先读本文件，再读 `progress.md`
